Arsitektur 8-Step Workflow

Kontribusi utama penelitian: alur kerja terstruktur dan terukur

1

Intent Classification

Klasifikasi input: chat (diskusi) vs task (pemrograman)

2

Agent Response

Acknowledgment singkat (1-2 kalimat) untuk konfirmasi pemahaman

3

Task Scheduler

Rencana tugas berbasis JSON dengan 2-6 langkah logis (ditampilkan sebagai tabel)

4-6

Action Iterations

Maksimal 5 iterasi, setiap iterasi: Thinking ‚Üí Execution ‚Üí Integrity Check

Thinking Phase

Penalaran internal (3-6 poin): tujuan, file target, edge cases, estimasi ukuran modifikasi

Action Execution

Eksekusi perintah (READ, WRITE, MODIFY, dll.) dengan batasan 500 baris/modifikasi

Integrity Check

Evaluasi hasil: skor 1-10, alasan (reasons), saran perbaikan (next_fix)

8

Final Summary

Ringkasan lengkap + 2-3 saran langkah berikutnya + pertanyaan konfirmasi
