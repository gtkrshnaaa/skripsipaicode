Arsitektur Single‑Shot Intelligence

Alur terstruktur: Intent → Acknowledgment → Planning JSON → Adaptive Execution → Finalization & Logging

1

Intent Classification

Klasifikasi input: chat (diskusi) vs task (pemrograman)

2

Dynamic Acknowledgment

Respons awal (1–2 kalimat) untuk mengonfirmasi pemahaman sebelum perencanaan

3

Planning JSON

Perencanaan terstruktur berbasis JSON: intent, berkas target, langkah aksi, kriteria sukses

4

Adaptive Execution

Eksekusi adaptif dalam 1–3 subfase memakai perintah workspace (READ, WRITE, MODIFY, TREE, LIST_PATH, MKDIR, TOUCH, RM, MV, FINISH) dengan batas diff-aware (≤500 baris, ≤50% rasio)

Finalization & Logging

Ringkasan hasil + saran langkah berikutnya. Pencatatan lengkap sesi ke .pai_history.

✓

Path Security & Diff‑Aware

Deny‑list direktori sensitif dan pembatasan diff (≤500 baris, ≤50%) dengan atomic write untuk mencegah korupsi.

Keunggulan Arsitektur Ini

Terstruktur: Alur jelas & dapat diaudit
Terencana: Planning JSON sebelum eksekusi
Terkontrol: Diff‑aware + atomic write
